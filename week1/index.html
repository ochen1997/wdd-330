Week 1 Notes

Some of the summaries of what I have gained through reading the material this week include the following,

  • When you write an application, you are communicating to the user.
  • For there to be true communication, the user needs some way to speak to you.
  • An application needs to be designed to store what is done, in order to be accessed when a device shuts down.
  • The information stored or shared with the user is called Output
  • Anything that the user enters is called Input.
  • When you want to get the user's input and textarea we use value
  • If the user wants to save anything and edit or add to it when they leave the page, the code that stores the information is localStorage.
  • JavaScript localStorage uses key-value pairs.
  • When making choices to talk to the user we use Comparison Operator.

When it comes to designing Content, we need to start with a mobile-first approach that will lead to Simple beautiful apps.

  • Mobile-first provides the best experience across many devices.
  • Address restriction to size/bandwidth.
  • Future Friendly and ready for growth.
