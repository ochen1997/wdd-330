Form validation include the following messages;

  • This field is required
  • Please enter your phone number in the format
  • Please enter a valid email address
  • Your password needs to be between 8 and 30 characters long and contain uppercase letters one symbol and a number.

The browser will and the web server wuill check to see that the data is in the correct format with the contraints set by the application.

The application allows the data to be submitted to the server usually the database

Reasons why form validation is required

  • We want to get the right data, in the right.
  • We want to protect our users data
  • We want to protect ourselves

Different types of client side form validaton include the following;

  • Built in form validation
  • JavaScript- validation is coded using JavaScript

Built in form validation include the following;

  • required - Specifies whether a form field needs to be filled in before the form can be submitted.
  • minlenght and maxlength - Specifies the minimumand maximum values of numerical input types.

When an element is valid, the following things are true;

  • The element matches the valid css pseudo-class
  • The user tries to send the data, the browser will submit the form

When an element is invalid, the following things are true;

  • The element matches the invalid css pseudo classes.
  • If the user tries to send the data, the browser will block the form and display an error.

You can constrain the character length of al text fields crdeated by input or textarea by using minlength and maxlength.

The fetch provides JavaScript interface for accessing and manipulating parts of the http pipeline.

To cause browsers to send a request wirh credentials included on both same origin and cross origin calls, add credentials inlude to init object as you pass to the fetch() method.

Files can be uploaded using an HTML input type= file input element, formatData and fetch.

Instead of passing a path to the resource you want to request into the fetch() call, you can create a request object using the Request() constructor, and pass that in as a fetch() method argument.

The headers interface allows you to create your own headers object via headers() constructor.
