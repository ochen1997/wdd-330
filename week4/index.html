The legacy DOM had a useful property called document.forms that returns an HTML collection of all the forms in the document in the order they appear in the markup. for example

  • const form = document.forms[0];
  • const form = document.getElementByTagname('form')[0];
  • Instead of using a numerical index, we can use the name attribute to identify a form eg; const form = document.forms.search;
  • const forms = document['search'];

A form objecy also has a method called elements that returns an HTML collection of all the elements contained in the form. in this case the form conatain two controls: an input element and a button element. for example;

  • const [input, button] = form.elements

we can also access the form controlls using their name attributes as if it was a property of the form object. for example.

  • const input = form.searchInput
  • const input = form['searchInput']

The form.submit() method will submit the form automatically.note that submitting aform using this method wont trigger the form submit event.

The form.reset() method will reset all the form controls back to their initial values specified in the HTML.

Object oriented programing is often used to model representation of objects in the real world. There are three main concepts in oops;

  • encapsulation
  • polymorphism
  • Inheritance

Encapsulation ivolves keeping all programming logic inside an object and making methods available to implement the functionallity.

polymorphism this means various objects can share the same methods but also have ability to override shared methods with a more specific implementation.

Inheritance this means we can take an object that already exists and inherit all its properties and methods. we can then improve on its functionallity by adding new properties.

class can be used to define a blue print for an object.

All objects have a constructor property thta returns the constructor function that created it.

The static keyword can be used in class declaration to create a static method.

JavaScript uses a prototypal inheritance model. This means that every class has a prototype property that is shared by every instance of the class.

An object instance can overwrite any properties or methods inherited from its prototype by simply assigning a new value to them.

The prototype can be used to add any new properties and methods after the class has been declared. it should be used to define any properties that will remain the same for every instance of the class.

By default, an object's methods are public in Javascript. Methods and properties are said to be public because they can be queried directly and changed by assignment.

Properties of objects in JavaScript are said to be enumerable or non enumerable. if they are not enumerable, this means they wll not show up when a for in loop is used to loop through an object's properties and methods.

A class can inherit from another class using the extends keyword in the class declaration.

The concept of polymorphism mean that different objects can have the same mehtod, but implement it in different ways.

It is possible to add more methods to the prototype of javaScript's built in objects such as numbers, string and array to add more functionallity. This practice is known as monkey-patching.

It is possible to avoid using classes altogether and create new objects bases on another object that ac as a blueprint or prototype instead.

Everytime a new object is created using the object.create() method, the new object inherits all the properties and methods from the parent objec, which becomes the new objects prototype.

It is possible to borrow methods from objects without having to inherit all their properties and methods. This is done by making a reference to the function that you want to borrow.

One of the most common uses of borrowing methods was to borrow methods from arrays.

Model view Controller is a design pattern thats been used for a long time in server side languages.

Models are object that implement the functionallity for creating, reading, updating and deleting specific pieces of information a bout the application as well as any other associated logic and behavior.

Views provide a visual representation of the model showing all the relevant information. in a web application, this would be the HTML displayed on the page. Views also provide a way for users to interact with an application usually via forms.

Controllers link models and views together by communicating between them. They respond to events, which are usuallyinputs from user, process the information then update models and view accordingly.

An MVC architecture can take a lot of code to implement and many frameworks have emerged that take care of much of the setup code for you.

One of the main features of MVC frameworks is data binding, which is the process of linking the model and view together.
